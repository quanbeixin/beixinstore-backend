# 发版检查清单

## 一、版本检查

- 目标版本 commit 已确认
- 发布范围和负责人已确认
- 回滚负责人已确认
- `package-lock.json` 已随版本提交

## 二、环境检查

- `.env.staging` 与 `.env.production` 键名一致
- `staging` 与 `production` Node.js 大版本一致
- `ENABLE_LEGACY_BOOTSTRAP` 在各环境保持关闭

## 三、数据库检查

- 本次结构变更已进入 `sql/migrations/`
- migration 文件命名符合时间与顺序规则
- migration 已先在 `staging` 验证通过
- 涉及破坏性变更时已准备回滚方案
- 正式数据库备份已完成

## 四、应用检查

- `npm ci` 执行成功
- 对应环境服务可正常启动
- `/health` 返回正常
- 关键登录流程验证通过
- 关键权限流程验证通过
- 核心业务接口验证通过
- 日志中无持续异常

## 五、发布后检查

- 服务进程状态正常
- 错误日志无新增异常堆栈
- 数据库连接数无异常上涨
- 登录、核心业务、查询与权限功能正常
- 如涉及前端联调，跨域配置验证通过
