# 文档目录

本目录用于存放项目的长期有效文档，重点记录接口约定、环境规则、发布规范和模块级设计说明，不记录一次性实施过程。

## 根目录文档

- `api.md`：后端接口总览与请求响应约定
- `environment-staging-production.md`：环境分层原则与 `staging / production` 边界
- `staging-rules.md`：本地与测试环境的使用、迁移与基础结构规则
- `release-sop.md`：发布标准流程与回滚原则
- `release-checklist.md`：发版检查清单
- `git-release-flow.md`：分支与版本流转规则

## 模块文档

- `project-management/`：项目管理模块的架构、数据库、接口、前端和开发规范

## 维护原则

- 只保留长期有效的原则性内容和规则性内容。
- 同类内容优先合并，不新增大量碎片化文档。
- 过程记录、单次排障、临时验证结果默认不进入文档。
- 后续新增文档默认使用中文记录。
