# 项目管理模块测试数据规则

## 一、文档目标

本文档仅定义测试数据规则，不记录排查步骤、执行步骤与单次处理过程。

## 二、测试数据原则

- 测试数据用于联调、演示与验收，不作为过程记录载体。
- 测试数据必须可识别、可复用、可清理。
- 测试数据应尽量贴近真实业务语义。

## 三、编码规则

- 中文写入统一使用 `utf8mb4`。
- 批量写入应通过脚本或 migration 执行，保证编码一致性。
- 禁止无编码声明的临时批量写入方式进入团队基线。

## 四、命名规则

- 项目、需求、缺陷命名必须具备业务可读性。
- 业务编码应使用可识别前缀（如 `PM-STAGE-*`）。
- 禁止长期保留无语义占位命名。

## 五、数据口径规则

- 状态、阶段、工时字段必须符合系统统一口径。
- 演示数据必须覆盖核心对象：业务线、需求池、缺陷、统计。
- 日志类演示数据必须保留完整操作语义。

## 六、边界规则

- 测试数据仅允许写入测试环境。
- 正式环境禁止写入未验证测试数据。
- 账号、权限、角色等关键数据优先通过 migration 维护。

## 七、文档维护规则

- 只记录测试数据长期规则与口径。
- 不记录单次 SQL、单次修复、单次验证结果。
