# 缺陷ID（bug_code）规则

## 目标

- 在缺陷管理中新增“缺陷ID（`bug_code`）”字段，作为业务可读编号。
- 与需求ID规则一致：支持手动指定，支持系统自动生成，格式固定，唯一约束。

## 规则定义

- 字段名：`pm_bugs.bug_code`
- 格式：`^BUG\\d{3,}$`（示例：`BUG001`、`BUG1024`）
- 唯一性：`uk_pm_bugs_bug_code` 唯一索引
- 创建时：
  - 传入合法 `bug_code`：按传入值写入
  - 未传入：系统自动生成下一个编号
- 更新时：
  - 允许修改为合法编号
  - 允许清空为 `NULL`

## 迁移说明

- 迁移脚本：`sql/migrations/20260324_004_bug_code.sql`
- 迁移动作：
  - 新增列 `bug_code`
  - 对历史数据按 `BUG + id(左补零)` 回填
  - 新增唯一索引 `uk_pm_bugs_bug_code`

## 接口与前端

- 后端缺陷接口支持 `bug_code` 字段读写。
- 缺陷列表支持显示 `bug_code`。
- 缺陷新建/编辑表单支持录入 `bug_code`，留空自动生成。
