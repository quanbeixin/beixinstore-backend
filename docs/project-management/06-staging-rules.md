# 项目管理模块测试环境规则

## 一、文档目标

本文档用于定义项目管理模块在 `staging` 环境下的长期规则，不记录排查步骤、实施步骤与单次迁移流水。

## 二、环境使用规则

- 项目管理模块默认在 `staging` 完成开发联调与验收。
- 未经验证的变更不得直接进入 `production`。
- 环境配置变更统一更新规则文档，不新增过程文档。

## 三、数据库变更规则

- 结构变更统一通过 migration 管理。
- migration 一经进入协作基线，不得随意改写历史版本。
- 关键数据结构与权限变更必须具备可重复执行能力。

## 四、联调规则

- 本地服务使用 `staging` 配置启动。
- 联调口径以测试环境数据库与权限数据为准。
- 功能联调、权限联调、业务线隔离联调必须在同一环境口径下完成。

## 五、账号与权限规则

- 测试环境账号仅用于测试与联调。
- 角色权限以测试库实际配置为唯一准入依据。
- 账号权限调整后必须同步更新规则文档。

## 六、数据边界规则

- 测试环境允许演示数据与联调数据。
- 正式环境禁止直接写入未验证测试数据。
- 临时测试数据不作为长期规则文档内容。

## 七、文档维护规则

- 仅保留环境边界、迁移口径、权限口径、发布顺序规则。
- 不记录单次迁移步骤、单次接口验证细节、单次排障过程。
