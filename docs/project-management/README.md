# 项目管理模块文档

本目录用于存放“项目管理系统”模块的长期有效文档，重点记录架构、规则、边界、口径和开发约定，不再沉淀阶段性过程记录。

## 当前文档

- `01-architecture.md`：模块架构设计，说明前端、后端、数据库的职责划分和模块边界
- `02-database-design.md`：数据库表结构设计、字段说明与初始化 SQL
- `03-backend-api-design.md`：后端 API 列表、路由设计、请求与响应约定
- `04-frontend-design.md`：前端页面设计、菜单结构、组件拆分与交互约定
- `05-development-rules.md`：项目管理模块开发规范，包含状态流转、权限、日志、统计和文档维护规则
- `06-staging-rules.md`：测试环境使用规范，包含联调原则、迁移规则和发布顺序
- `07-test-data-rules.md`：测试数据规范，包含中文写入、命名方式和乱码排查顺序

## 维护原则

- 只保留长期有效的原则性内容和规则性内容。
- 不记录“第几批实现”“某次排障过程”“逐步验证流水”等过程型内容。
- 同类内容优先合并到现有主题文档中，避免文档越拆越碎。
- 后续新增文档默认使用中文记录。
